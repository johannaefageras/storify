# Tone: Quest Log

## Overview

The quest log tone. The day is transformed into an RPG adventure, complete with quests, XP gains, loot drops, stats, achievements, and boss battles. Every mundane activity becomes a mission objective, every meal is a consumable item with stats, and every social interaction affects reputation. The humor comes from applying epic gaming language to completely ordinary life. Think World of Warcraft meets Skyrim meets Borderlands meets Pok√©mon ‚Äì the universal language of video games applied to a Tuesday in Gothenburg.

## Voice & Style

### Core Characteristics

- **Second/Third person game narration** ("You," "Player," or character name)
- **Game UI elements** ‚Äì progress bars, stats, notifications, tooltips
- **Quest/mission structure** ‚Äì objectives, completion status, rewards
- **Item drops with rarity** ‚Äì Common, Uncommon, Rare, Epic, Legendary
- **Buffs and debuffs** ‚Äì status effects from life events
- **Achievement popups** ‚Äì celebrating small victories

### Sentence Structure

- Short, punchy game notifications
- Bracketed tags: [QUEST COMPLETE], [NEW ITEM], [ACHIEVEMENT]
- Progress indicators: ‚ñà‚ñà‚ñà‚ñà‚ñà‚ñà‚ñë‚ñë‚ñë‚ñë 60%
- Stat changes: +15 mood, -10 energy
- Tooltips in italics for item descriptions
- Mix of narrative and UI elements

### Vocabulary

- Quest terms: objective, mission, complete, failed, in progress, reward
- Stats: HP, XP, energy, stamina, mood, reputation, skill level
- Items: loot, drop, consumable, equipped, inventory, rarity
- Combat: encounter, boss, defeated, critical hit, dodge, survive
- Progress: level up, achievement unlocked, checkpoint, save point
- Buffs/debuffs: buff, debuff, status effect, cooldown, expired

## Structure

### Opening

Begin with a game-style day header. Options:

- Daily log format: "**DAILY LOG ‚Äì Day [X] ‚Äì [WEEKDAY]**"
- Quest start: "**[NEW QUEST AVAILABLE]** Survive Tuesday"
- Loading screen: "**LOADING...** Wednesday // _Tip: Coffee increases reaction time by 12%_"
- Player status: "**PLAYER STATUS** ‚Äì Morning ‚Äì Energy: ‚ñà‚ñà‚ñë‚ñë‚ñë‚ñë‚ñë‚ñë‚ñë‚ñë 20%"

### Body

1. **Morning phase** ‚Äì Wake up sequence, breakfast consumables, departure
2. **Main quest** ‚Äì School/primary activity as mission objectives
3. **Random events** ‚Äì Unexpected things framed as rare spawns/events
4. **Side quests** ‚Äì Social interactions, small moments
5. **Loot drops** ‚Äì Food, items, nice moments as acquired items
6. **Boss battles** ‚Äì Challenges faced and overcome (or not)

### Closing

End with one of:

- End of day stats summary
- Progress saved notification
- Loading screen tip for tomorrow
- Achievement summary
- "Continue? [YES] [NO]"

## Tone Guidelines

### DO ‚úì

- Use visual elements: progress bars, checkboxes, stat numbers
- Apply item rarity to everything (Legendary pasta, Common homework)
- Track stats throughout the day (energy, mood, social)
- Include achievement unlocks for small wins
- Frame challenges as boss fights or difficult encounters
- Add tooltips/flavor text for items
- Mix different game genres (RPG, combat, achievement hunting)
- Include loading screen tips
- Use gaming humor and references

### DON'T ‚úó

- Make it too complicated or hard to read
- Use only one game's specific terminology (keep it universal)
- Forget the actual events ‚Äì they're the content!
- Make it feel like a chore ‚Äì this should be FUN
- Overdo the stats (some, not constant)
- Be negative about "failed" quests ‚Äì keep it light
- Use super obscure gaming references
- Lose the narrative in too much UI

## Variation Notes

To keep entries from feeling repetitive over time:

- Vary the game genre emphasis (more RPG, more achievement-hunter, more combat)
- Rotate which elements appear (not every entry needs all elements)
- Change the opening format (loading screen, quest start, status check)
- Vary item rarities based on day quality
- Include different achievement types
- Sometimes focus more on one big "boss battle," sometimes many small quests
- Play with different stat categories
- Vary the loading screen tips
- Change closing format (save progress, continue screen, checkpoint)
- Occasionally include "secret achievements" or "hidden quests"
- Let some days be "grinding" days with steady XP, others "raid" days with big events
